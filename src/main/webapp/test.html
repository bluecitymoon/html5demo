Basic Page Setup

With HTML 5 applied to our theme, the doctype is now only <!DOCTYPE html> . All of the IE conditionals not specific to an IE only style sheet will reside before the <head> tag for speedier loading. The meta viewport must be applied to create the proper ratio when viewing on mobile devices. More IE specific attention is required on a specific style sheet before the closing </head> tag.

The javascript files should reside below the footer section and before the closing body tag. There is an exception for FOUC and you can place the problem javascript immediately before the closing head tag. 

<!DOCTYPE html>
<!--[if lte IE 7]> <html class="no-js lt-ie9 lt-ie8" lang="en"> <![endif]-->
<!--[if IE 8]> <html class="no-js lt-ie9" lang="en"> <![endif]-->
<!--[if gt IE 8]> <html class="no-js ie9" lang="en"> <! --> <html lang="en"> <!--<![endif]-->
<head>
	<meta charset="utf-8" />
	<!-- Set the viewport width to device width for mobile -->
	<meta name="viewport" content="width=device-width" />
	<title>Werner Enterprises</title>
	<!-- Required CSS Files -->
	<link rel="stylesheet" href="https://static.werner.com/werner/2.3/css/styles.min.css">
	<link rel="stylesheet" href="https://static.werner.com/werner/2.3/css/foundation/print.css" media="print">
	<!--[if IE]>
	<link rel="stylesheet" href="https://static.werner.com/werner/2.3/css/foundation/ie.css">
	<script src="https://static.werner.com/werner/2.3/js/html5.js"></script>
	<![endif]-->
</head>
<body>


	<!-- [ Header, Body Content, and Footer Code will go here. ] -->


	<!-- Required JS Files -->
	<script src="https://static.werner.com/werner/2.3/js/jquery-1.8.3.min.js"></script>
	<script src="https://static.werner.com/werner/2.3/js/jquery-ui-1.8.23.min.js"></script>
	<script src="https://static.werner.com/werner/2.3/js/werner-theme.js"></script>
	<!-- jQuery Modules -->
	<!-- [ Place any jQuery modules or extra javascript here. ] -->
	<script>
		$(document).ready(function(){

			// -- [ Page specific javascript goes here. ] --

		});
	</script>
</body>
</html>

Site Navigation Updated in 2.3

To show an active state for the main navigation heirarchy the class active must be added to the anchor. Otherwise, the regular links are just a <li><a href=""></a></li> without a class or id attribute.

If you will not be using the sub-navigation links, remove the class sub-nav from the nav tag.

Content Wrapper

The Content wrapper is probably the most important part of the aesthetics in the enterprise theme. This encompasses the entire pagee content. By wrapping the application in this code, it creates a white background, proper page padding, and sets up the foundation for the grid system. This code should be placed immediately after the navigation segment.

<section class="mainbg">

Advanced Search and Tools

This is a two part module. The text that has the toggle function applied to it and the content that is shown after the toggle. A jQuery initialization is required for this to work.

Advanced Search and Tools

Advanced Search

Content...

<div class="row">
	<div class="twelve columns">
		<p>
			<a id="advsearch-toggle" class="toolsarrow">Advanced Search and Tools</a>
		</p>
	</div>
</div>

Toggled Content: Wrap all content that will be toggled with the code below. This will be placed directly underneath the Search and Tools code. 

<div class="row" id="advsearch">
	<form class="nice custom">	
		<fieldset><legend>Advanced Search</legend>
				<p>Content...</p>
		</fieldset>			
	</form>
</div><!-- // end advanced search and tools -->

Initializing: Add this javascript to activate the Advanced Search and Tools toggle. It should be placed inside the $(document).ready() function. 

// Advanced Search & Tools Toggle
$('#advsearch').hide();
$('#advsearch-toggle').click(function(){
	$('#advsearch').slideToggle(400);
	$(this).toggleClass("toolsarrow-active");
	return false;
});

App Wrapper

The main function of the app wrapper is to give a work area based on the number of columns you choose (up to 12). You can have more than one app wrapper. The wrapper also places a 1pt gray line around the module area. This cannot be removed. If you are going to use a module header with the wrapper, you will need to apply the class ui-corner-top . This adds round corners to just the top of the wrapper, hiding it behind the header module. 

<div class="row">
	<div class="eight columns">
		<div class="app-wrapper ui-corner-top">

			<!-- [ Module Headers and Content ] -->

		</div>
	</div>
</div>
